# P测试阈值设置说明

## 基本原则

P测试（Phenomenon-Level Tests）的阈值设置遵循以下原则：

1. **现象存在性验证**：检测心理效应是否存在，而非精确匹配数值
2. **容错性**：允许一定比例的场景/项目不显示效应（考虑统计噪音）
3. **原文依据**：基于原始论文中效应的稳健性和一致性

---

## Study 001 的P测试阈值

### P1: Study 1 (4个假设场景) - 阈值 75%

```json
{
  "test_id": "P1",
  "method": {
    "test": "proportion_test",
    "scenarios": ["study_1_supermarket", "study_1_term_paper", 
                  "study_1_traffic_ticket", "study_1_space_program"],
    "min_proportion": 0.75
  }
}
```

**设置理由**：
- **原文结果**：4个场景全部显示显著FCE（p < 0.001）
- **效应大小**：范围从8.9%到21.5%，最小的space_program也是正向
- **阈值75%**：要求至少3/4场景通过
  - ✅ 允许1个场景失败（考虑LLM的随机性）
  - ✅ 仍然保证大多数场景显示效应
  - ✅ 避免过于严格（100%可能因单个场景失败而整体失败）

**为什么不是100%？**
- LLM可能在某些特定场景上表现不稳定
- 小样本情况下统计噪音可能导致单个场景FCE为负
- 75%是"大多数"的合理标准

### P2: Study 2 (34项问卷) - 阈值 60%

```json
{
  "test_id": "P2",
  "method": {
    "test": "proportion_test",
    "min_proportion": 0.6
  }
}
```

**设置理由**：
- **原文结果**：34个项目中32个显示预期方向（94%）
- **项目多样性**：涵盖个人特质、偏好、期望等多个维度
- **阈值60%**：要求至少20/34项目通过
  - ✅ 比Study 1更宽松（因为项目更多更多样）
  - ✅ 仍然是明确的"多数"标准
  - ✅ 允许约1/3的项目不显示效应

**为什么比P1低？**
- **项目数量多**：34个项目 vs 4个场景
- **内容多样性**：不同类型的个人特质，效应强度可能差异大
- **原文也有例外**：原文中也有2个项目（6%）未显示预期方向
- **统计考虑**：项目越多，越可能有个别项目因随机性失败

**为什么是60%而不是50%？**
- 50%只是"一半"，不足以证明效应的系统性
- 60%是明确的"大多数"，符合"超过半数"的直觉
- 与原文94%的成功率相比，60%已经是很宽松的标准

### P3: Study 3 (2个标语实验) - 阈值 100%

```json
{
  "test_id": "P3",
  "method": {
    "test": "proportion_test",
    "scenarios": ["study_3_sign_joes", "study_3_sign_repent"],
    "min_proportion": 1.0
  }
}
```

**设置理由**：
- **原文结果**：2个场景都显示极强的FCE（33.4%和28.6%）
- **效应强度**：是Study 1的2-3倍
- **场景数量少**：只有2个场景
- **阈值100%**：要求2/2场景都通过
  - ✅ 场景太少，不能允许失败
  - ✅ 效应非常强，应该稳定复现
  - ✅ 如果连2个都不能全过，说明效应不够稳健

**为什么必须100%？**
- **只有2个场景**：如果允许1个失败，就只剩50%，无法证明效应
- **效应极强**：原文中FCE是Study 1的2-3倍，应该很容易检测
- **真实行为**：Study 3是真实行为选择（不是假设），效应应该更强

## 阈值设置的一般规则

### 1. 场景/项目数量

| 数量 | 建议阈值 | 理由 |
|------|----------|------|
| 2-3个 | 100% | 太少，不能允许失败 |
| 4-10个 | 75-80% | 允许1-2个失败 |
| 10-30个 | 60-70% | 允许约1/3失败 |
| 30+个 | 50-60% | 允许接近一半失败 |

### 2. 效应强度

| 原文效应 | 建议阈值调整 |
|----------|--------------|
| 极强 (>30%) | 提高10-20% |
| 中等 (10-30%) | 标准阈值 |
| 较弱 (<10%) | 降低10-20% |

### 3. 内容一致性

| 内容类型 | 建议阈值调整 |
|----------|--------------|
| 高度一致（同类场景） | 提高阈值 |
| 中等多样（相关场景） | 标准阈值 |
| 高度多样（不同维度） | 降低阈值 |

## 其他研究的P测试阈值示例

### Study 002 (Anchoring)
```json
{
  "test_id": "P1",
  "description": "Overall Anchoring Effect",
  "method": {
    "test": "one_sample_t_test",
    "comparison_value": 0,
    "alternative": "greater"
  }
}
```
- **不使用比例阈值**：因为测试的是整体Anchoring Index
- **使用t检验**：检验平均值是否显著大于0
- **阈值隐含在p<0.05中**：统计显著性本身就是阈值

### Study 003 (Framing)
```json
{
  "test_id": "P1-P10",
  "description": "Framing effect for each problem",
  "method": {
    "test": "chi_square",
    "alpha": 0.05
  }
}
```
- **每个问题独立测试**：10个P测试（P1-P10）
- **阈值为p<0.05**：每个问题必须显示显著的框架效应
- **全部必须通过**：因为原文中10个问题都显示了显著效应

### Study 004 (Representativeness)
```json
{
  "test_id": "P1-P8",
  "description": "Representativeness bias for each problem",
  "method": {
    "test": "proportion_test",
    "min_proportion": varies
  }
}
```
- **不同问题不同阈值**：
  - 选择题类问题：检测选择偏向比例
  - 估计类问题：检测估计偏差方向

## 阈值设置的决策流程

```
1. 查看原文结果
   ├─ 所有场景/项目都显著？ → 考虑高阈值（80-100%）
   └─ 有少数例外？ → 考虑中等阈值（60-75%）

2. 考虑场景/项目数量
   ├─ ≤3个 → 倾向100%
   ├─ 4-10个 → 倾向75-80%
   └─ >10个 → 倾向60-70%

3. 考虑效应强度
   ├─ 极强 → 提高阈值
   ├─ 中等 → 标准阈值
   └─ 较弱 → 降低阈值

4. 考虑内容一致性
   ├─ 高度一致 → 提高阈值
   └─ 高度多样 → 降低阈值

5. 最终决策
   └─ 在上述因素间平衡，确保阈值既有挑战性又合理
```

## 关键要点

1. **P测试阈值不是随意设置的**
   - 基于原文效应的稳健性
   - 考虑场景/项目数量
   - 平衡严格性和合理性

2. **阈值应该有挑战性但可达到**
   - 太低（如30%）：无法证明效应的系统性
   - 太高（如100%用于多场景）：可能因个别失败而整体失败

3. **不同研究的阈值可以不同**
   - 取决于原文效应的特性
   - 取决于测试设计的目标

4. **阈值设置是领域知识的体现**
   - 需要理解原文的实验设计
   - 需要判断效应的预期稳健性
