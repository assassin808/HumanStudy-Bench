步骤1: 查看场景/项目数量
│
├─ 2-3个场景
│  │
│  ├─ 原文效应极强 (>30%)
│  │  └─→ 阈值: 100% ✓ (Study 001 P3)
│  │     理由: 场景少且效应强，必须全部通过
│  │
│  └─ 原文效应中等 (10-30%)
│     └─→ 阈值: 80-100%
│        理由: 场景少，最多允许1个失败
│
├─ 4-10个场景
│  │
│  ├─ 原文全部显著 (100%)
│  │  └─→ 阈值: 75-80% ✓ (Study 001 P1)
│  │     理由: 允许1-2个失败，保持大多数通过
│  │
│  └─ 原文有少数例外 (80-95%)
│     └─→ 阈值: 70-75%
│        理由: 与原文成功率保持一致
│
└─ 10+个场景/项目
   │
   ├─ 内容高度一致
   │  └─→ 阈值: 70-75%
   │     理由: 一致性高，期望更多通过
   │
   └─ 内容高度多样 ✓ (Study 001 P2)
      └─→ 阈值: 60-70%
         理由: 多样性高，允许更多失败

Study 001 具体案例分析
═══════════════════════════════════════════════════════════════════

P1: Study 1 (4个假设场景)
┌─────────────────────────────────────────────────────────────┐
│ 输入特征:                                                    │
│   • 场景数量: 4个                                            │
│   • 原文成功率: 100% (4/4全部显著)                          │
│   • 效应强度: 中等 (8.9% - 21.5%)                           │
│   • 内容一致性: 高 (都是假设情境中的选择)                   │
│                                                              │
│ 决策路径:                                                    │
│   4-10个场景 → 原文全部显著 → 阈值75-80%                   │
│                                                              │
│ 最终阈值: 75% (3/4必须通过)                                 │
│                                                              │
│ 解释:                                                        │
│   • 允许1个场景失败 (考虑LLM随机性)                         │
│   • 仍然要求"大多数"场景通过                                │
│   • 与原文100%成功率相比已经宽松                            │
└─────────────────────────────────────────────────────────────┘

P2: Study 2 (34项问卷)
┌─────────────────────────────────────────────────────────────┐
│ 输入特征:                                                    │
│   • 项目数量: 34个                                           │
│   • 原文成功率: 94% (32/34显示预期方向)                     │
│   • 效应强度: 中等 (平均10-15%)                             │
│   • 内容一致性: 低 (涵盖多种个人特质和偏好)                 │
│                                                              │
│ 决策路径:                                                    │
│   10+个项目 → 内容高度多样 → 阈值60-70%                    │
│                                                              │
│ 最终阈值: 60% (20/34必须通过)                               │
│                                                              │
│ 解释:                                                        │
│   • 项目多且多样，允许约1/3失败                             │
│   • 60%是明确的"多数"标准                                   │
│   • 比原文94%宽松很多，考虑LLM的不确定性                    │
└─────────────────────────────────────────────────────────────┘

P3: Study 3 (2个标语实验)
┌─────────────────────────────────────────────────────────────┐
│ 输入特征:                                                    │
│   • 场景数量: 2个                                            │
│   • 原文成功率: 100% (2/2都显示极强FCE)                     │
│   • 效应强度: 极强 (28.6% - 33.4%)                          │
│   • 内容一致性: 高 (都是标语佩戴实验)                       │
│                                                              │
│ 决策路径:                                                    │
│   2-3个场景 → 原文效应极强 → 阈值100%                      │
│                                                              │
│ 最终阈值: 100% (2/2必须通过)                                │
│                                                              │
│ 解释:                                                        │
│   • 场景太少，不能允许失败                                  │
│   • 效应极强，应该稳定复现                                  │
│   • 如果50%通过无法证明效应的系统性                         │
└─────────────────────────────────────────────────────────────┘

阈值设置的权衡
═══════════════════════════════════════════════════════════════════

太严格 (接近100%)
  优点: ✓ 确保效应非常稳健
  缺点: ✗ 可能因个别失败而整体失败
        ✗ 不考虑LLM的随机性和不确定性
        ✗ 可能过于苛刻

适中 (60-80%)
  优点: ✓ 平衡严格性和合理性
        ✓ 允许一定的失败率
        ✓ 仍然证明效应的系统性
  缺点: ✗ 需要仔细设计阈值

太宽松 (接近50%)
  优点: ✓ 容易通过
  缺点: ✗ 无法证明效应的系统性
        ✗ 50%可能只是随机
        ✗ 失去测试的意义

关键原则
═══════════════════════════════════════════════════════════════════

1. 基于原文 (Evidence-Based)
   └─ 查看原文中效应的稳健性和一致性

2. 考虑数量 (Quantity-Aware)
   └─ 场景/项目越多，阈值可以越低

3. 考虑强度 (Strength-Aware)
   └─ 效应越强，阈值应该越高

4. 考虑多样性 (Diversity-Aware)
   └─ 内容越多样，阈值可以越低

5. 保持合理性 (Reasonableness)
   └─ 阈值应该有挑战性但可达到
